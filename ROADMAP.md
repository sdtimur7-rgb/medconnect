# Roadmap MedConnect

План развития платформы по этапам.

## ✅ Этап 1: MVP (Текущий)

**Статус:** Завершён

- [x] Docker Compose инфраструктура
- [x] PostgreSQL схема БД (5 таблиц)
- [x] FastAPI с базовыми эндпоинтами
- [x] Telegram бот (aiogram 3)
- [x] Celery планировщик задач
- [x] Интеграции (1С, SMS, Chatwoot)
- [x] ИИ-менеджер (Claude API)
- [x] Биллинг модуль

## 🚧 Этап 2: Интеграция с 1С (Неделя 1-2)

**Цель:** Полноценная синхронизация с реальным центром

- [ ] Адаптация клиента 1С под конкретную конфигурацию
- [ ] Маппинг полей и форматов данных
- [ ] Обработка edge cases (нет телефона, отменённые записи)
- [ ] Тесты на реальных данных
- [ ] Логирование всех синхронизаций

**Критерий готовности:** Записи автоматически подгружаются каждые 15 мин.

## 📱 Этап 3: Напоминания (Неделя 2-3)

**Цель:** Работающий флоу напоминаний

- [ ] Реализация отправки напоминаний за 24ч
- [ ] Реализация отправки напоминаний за 2ч
- [ ] SMS дублирование через СМСЦентр
- [ ] Обработка подтверждений через inline кнопки
- [ ] Создание billing_events при подтверждении
- [ ] Уведомления в Chatwoot

**Критерий готовности:** Пациенты получают напоминания и могут подтвердить.

## 🤖 Этап 4: ИИ-менеджер (Неделя 3-4)

**Цель:** ИИ отвечает на типовые вопросы

- [ ] Настройка system prompt под центр
- [ ] Обработка входящих сообщений через Claude
- [ ] Определение сложных вопросов (needs_human)
- [ ] Форвардинг в Chatwoot
- [ ] История диалогов
- [ ] Метрики ответов ИИ

**Критерий готовности:** ИИ отвечает на 70%+ вопросов без менеджера.

## 💰 Этап 5: Биллинг и отчёты (Неделя 4-5)

**Цель:** Автоматическая генерация счетов

- [ ] Ежемесячная генерация invoices
- [ ] PDF отчёты с детализацией
- [ ] Email рассылка счетов
- [ ] Dashboard статистики для центра
- [ ] Экспорт данных (CSV/Excel)

**Критерий готовности:** 1го числа приходит счёт на email.

## 🎨 Этап 6: Web панель администратора (Неделя 5-6)

**Цель:** UI для управления платформой

- [ ] Frontend на React
- [ ] Список центров и их настройки
- [ ] Статистика по подтверждениям
- [ ] Просмотр логов синхронизаций
- [ ] Управление тарифами
- [ ] Генерация отчётов on-demand

**Технологии:** React + TypeScript + Tailwind CSS

## 📊 Этап 7: Аналитика (Неделя 7-8)

**Цель:** Insights для центров

- [ ] Процент подтверждений по времени суток
- [ ] Процент no-show по врачам/специальностям
- [ ] Эффективность напоминаний (Telegram vs SMS)
- [ ] ROI платформы для центра
- [ ] Графики и визуализация
- [ ] Экспорт отчётов

## 🚀 Этап 8: Масштабирование (Неделя 9-10)

**Цель:** Готовность к 10+ центрам

- [ ] Multi-tenancy изоляция данных
- [ ] Rate limiting для API
- [ ] Кэширование через Redis
- [ ] Очереди сообщений (RabbitMQ)
- [ ] Мониторинг (Prometheus + Grafana)
- [ ] Alerting (PagerDuty/Opsgenie)

## 🛡️ Этап 9: Безопасность

**Цель:** Production-ready безопасность

- [ ] SSL сертификаты (Let's Encrypt)
- [ ] JWT аутентификация для API
- [ ] RBAC (Role-Based Access Control)
- [ ] Шифрование чувствительных данных
- [ ] Аудит логи
- [ ] GDPR compliance
- [ ] Бэкапы в S3

## 📱 Этап 10: Мобильное приложение (Опционально)

**Цель:** Native apps для менеджеров

- [ ] Flutter/React Native app
- [ ] Push уведомления
- [ ] Оффлайн режим
- [ ] Быстрые ответы пациентам
- [ ] Статистика в реальном времени

## 🌐 Этап 11: Дополнительные интеграции

**Расширение функционала:**

- [ ] Интеграция с МИС (Медицинскими информационными системами)
- [ ] Яндекс.Карты для навигации до центра
- [ ] WhatsApp Business API
- [ ] Email напоминания
- [ ] Telegram Mini App для записи
- [ ] Онлайн оплата консультаций

## 📈 KPI для оценки успеха

### Для платформы:
- Количество активных центров
- Количество отправленных напоминаний
- Процент подтверждений (цель: >60%)
- Uptime сервисов (цель: >99.5%)

### Для центров:
- Снижение no-show (цель: -30%)
- Экономия времени регистратуры (цель: 10 часов/месяц)
- ROI платформы (экономия vs стоимость)

## Приоритеты

**P0 (критично):**
- Этапы 2-5 (интеграция, напоминания, биллинг)

**P1 (важно):**
- Этапы 6-7 (админка, аналитика)

**P2 (желательно):**
- Этапы 8-11 (масштабирование, доп. функции)
